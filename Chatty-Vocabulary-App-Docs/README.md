# 🗣 Chatty Vocabulary App: Полный Комплект Документации Требований

## 🚀 Обзор Проекта

Данный репозиторий содержит полный комплект артефактов, разработанных в процессе **сбора, анализа и формализации требований** к мобильному приложению **Chatty** для изучения словарного запаса.

Проект демонстрирует глубокие навыки в области **Software Requirements Specification (SRS)**, **моделирования пользовательского взаимодействия (Use Case)** и применения нотаций UML для создания однозначной технической документации.

## 🎯 Ключевые Компетенции и Задачи

1.  **Создание Спецификации Требований (SRS):** Разработка детального документа, описывающего функциональные и нефункциональные требования к ПО в соответствии со стандартами.
2.  **Моделирование Use Case:** Идентификация, описание и визуализация ключевых пользовательских сценариев (Use Cases), таких как работа с карточками и словарем.
3.  **Применение UML Нотаций:** Использование UML для создания словарей и диаграмм, обеспечивающих единое понимание терминологии и структуры системы.
