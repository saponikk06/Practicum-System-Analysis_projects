# 🗣 Chatty Vocabulary App: Полный Комплект Документации Требований

**Теги:** `System Analysis` `Requirements Engineering` `SRS` `UML` `Use Case` `Draw.io`

## 🚀 Обзор Проекта

Данный проект содержит полный комплект артефактов, разработанных в процессе **сбора, анализа и формализации требований** к мобильному приложению **Chatty** для изучения словарного запаса.

Проект демонстрирует навыки в области разработки **Software Requirements Specification (SRS)**, **моделирования пользовательского взаимодействия (Use Case)** и применения нотаций UML для создания однозначной технической документации, готовой для передачи в разработку.

## 🎯 Ключевые Компетенции и Задачи

1.  **Создание Спецификации Требований (SRS):** Разработка детального документа, описывающего функциональные и нефункциональные требования к ПО в соответствии с отраслевыми стандартами.
2.  **Моделирование Use Case:** Идентификация, описание и визуализация ключевых пользовательских сценариев, таких как работа с флеш-карточками и словарем.
3.  **Применение UML Нотаций:** Использование UML для создания словарей и диаграмм, обеспечивающих единое понимание терминологии и структуры системы между бизнесом и командой разработки.

## 🛠 Используемые Нотации и Инструменты

* **Документирование Требований:** SRS (Software Requirements Specification), Use Case Specifications.
* **Моделирование:** UML (Unified Modeling Language), Use Case Diagram.
* **Инструмент для Диаграмм:** Draw.io (diagrams.net).

## ⚙️ Системные требования и инструкция по просмотру

Проект состоит из текстовой документации и графических схем. Развертывание среды выполнения не требуется.
* Файлы спецификаций представлены в формате `.docx` (требуется Microsoft Word, Google Docs или LibreOffice Writer).
* Изображения диаграмм доступны в формате `.jpg` и открываются стандартными средствами ОС или в браузере.
* Для редактирования или детального изучения исходников диаграмм (`.drawio`) необходимо использовать десктопное приложение Draw.io или веб-версию [diagrams.net](https://app.diagrams.net/).

## 📈 Выводы по результатам проекта

В результате работы над проектом был сформирован исчерпывающий пакет аналитической документации для мобильного приложения Chatty. Формализация требований в виде SRS позволила четко очертить границы системы, выделить функциональные (работа с карточками, алгоритмы повторения) и нефункциональные (производительность, безопасность) ограничения. Визуализация Use Case и составление словарей данных устранили потенциальные неоднозначности в трактовке терминов. Проект полностью готов к этапам архитектурного проектирования и разработки.
