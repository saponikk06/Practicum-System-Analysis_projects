# Academic-Works-SQL-DB 💾📚

## Обзор Проекта ✨

Данный проект представляет собой реляционную базу данных, спроектированную для хранения и управления академическими работами, статьями, патентами и исследованиями. 🤓 Система **Academic-Works-SQL-DB** обеспечивает централизованный учет всех связанных метаданных: авторов, их аффилиаций, ключевых слов, журналов и данных о цитировании. 📊 Проект демонстрирует навыки концептуального и логического моделирования данных, а также создания оптимизированной схемы БД.

## Ключевые Цели и Задачи 🎯

Проектирование и реализация базы данных преследовали следующие цели, обеспечивающие эффективную работу с научным контентом:

1.  **Концептуальное Моделирование:** Разработка логической модели данных (ERD) для определения сущностей и связей, необходимых для учета академических работ и авторов. 🔗
2.  **Спецификация Данных:** Создание полного словаря данных с описанием всех атрибутов, их типов и ограничений для обеспечения целостности информации. 📝
3.  **Оптимизация запросов:** Разработка схемы БД с учетом требований нормализации и индексирования для поддержки быстрых аналитических и поисковых SQL-запросов. ⚡️
4.  **Скрипты развертывания:** Создание SQL-скриптов для быстрого развертывания базы данных, включая команды `CREATE TABLE` и `INSERT` для наполнения тестовыми данными. ⚙️

## Структура Репозитория и Артефакты 📂

Проект структурирован по этапам моделирования и реализации. Все ключевые артефакты, подтверждающие экспертизу в проектировании БД, находятся в соответствующих файлах:

| Папка / Файл | Ключевые Артефакты | Описание |
| :---: | :---: | :---: |

| `03_SQL-Scripts` | `create_schema.sql` | Скрипт SQL, содержащий команды для создания всех таблиц и связей (`CREATE TABLE`). 🏗️ |

## Пример SQL-запроса (Поиск работ по ключевому слову) 🔍

```sql
SELECT
    t1.title,
    t1.publication_date
FROM
    Works t1
JOIN
    Work_Keyword t2 ON t1.work_id = t2.work_id
WHERE
    t2.keyword_text = 'Биоинформатика';
