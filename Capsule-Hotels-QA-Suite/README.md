# Тестирование сервиса онлайн-бронирования капсульных отелей «Капсула»

Проект `Capsule-Hotels-QA-Suite` содержит комплексную тестовую и пользовательскую документацию для веб-приложения сети капсульных отелей. Артефакты разработаны для верификации функциональных требований, проверки исправленных дефектов и подготовки продукта к релизу (версия 2.2).

## 📌 Описание проекта
Объект тестирования — электронная система резервирования номеров «Капсула». Приложение предоставляет пользователям функционал для поиска отелей на интерактивной карте, регистрации, оформления бронирований и управления личным кабинетом. Цель данного QA-проекта — контроль качества программного продукта и подтверждение его соответствия техническому заданию (требования TR-1.1 – TR-5.1).

## 🛠 Методология и компетенции
* **Виды тестирования:** Функциональное тестирование, регрессионное тестирование (проверка после спринта исправлений), тестирование пользовательского интерфейса.
* **Артефакты QA:** Разработка Test Plan, Test Cases, написание программы и методики испытаний, составление User Manual.

## 📂 Структура проекта
Содержимое проекта строго разделено по целевым документам:

* **`Test_Plan_Capsule_v2.2.pdf`** — План тестирования. Содержит матрицу результатов, детализированные тест-кейсы (ID: Capsule-1 — Capsule-12) с описанием шагов, тестовых данных, ожидаемых и фактических результатов. Описывает как успешные сценарии, так и найденные баги.
* **`Test_Program_Methodology_Capsule.pdf`** — Программа и методика испытаний. Регламентирующий документ, описывающий объект и цели испытаний, а также детальные сценарии проверок авторизованной зоны (история бронирований, смена контактной информации).
* **`User_Manual_Capsule.pdf`** — Руководство пользователя. Пошаговая инструкция по эксплуатации системы: от обзора удобств отеля до оформления бронирования. Включает важный раздел «Сообщения оператору» с разбором клиентских ошибок (некорректный email, ошибки формата даты/времени).

## 🚀 Основной покрытый функционал
Тестовые сценарии покрывают следующие ключевые модули:
1. **Навигация и UI:** Отображение адресов и ближайших станций метро через всплывающие подсказки на интерактивной карте.
2. **Пользовательские сессии:** Регистрация, авторизация (включая обработку ошибок входа), сброс пароля.
3. **Личный кабинет:** Управление профилем, редактирование email/телефона, просмотр истории операций.
4. **Управление бронированиями:** Создание брони (с авторизацией и без), отмена существующих бронирований, проверка бизнес-логики выбора дат и времени.
