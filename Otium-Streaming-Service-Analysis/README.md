# 📺 Otium Streaming Service: Анализ и Спецификация Функционала

**Теги:** `System Analysis` `UML Sequence Diagram` `Technical Specification` `Dev Ticket` `Draw.io` `Requirements Engineering`

## 🚀 Обзор Проекта

Данный проект содержит ключевые артефакты детального анализа и проектирования для **потокового видеосервиса Otium**. Фокус проекта — на моделировании взаимодействия между компонентами системы и создании формальной технической документации для внедрения конкретной функции.

Проект демонстрирует навыки глубокого анализа, применения нотаций UML для моделирования поведения системы и оформления требований в формате, готовом для передачи в разработку (Dev Ticket).

## 🎯 Ключевые Цели и Задачи

1.  **Моделирование Взаимодействия:** Разработка Диаграммы Последовательности (Sequence Diagram) для детальной визуализации процесса взаимодействия пользователя с сервисом при выборе контента.
2.  **Техническая Спецификация:** Формализация требований к новой функции "Фильтрация по эксклюзивному контенту" в виде **Тикета на разработку (Dev Ticket/Tech Spec)**, включая технические и функциональные детали.
3.  **Трассировка Требований:** Обеспечение четкой связи между аналитическим моделированием и финальным заданием для команды разработки.

## 🛠 Используемые Технологии и Нотации

* **Моделирование Взаимодействия:** UML Sequence Diagram (Диаграмма Последовательности)
* **Инструментарий:** Draw.io (diagrams.net)
* **Документация:** MS Word (Тикет на разработку)

## ⚙️ Системные требования и инструкция по просмотру

Проект содержит аналитическую документацию и UML-схемы. Развертывание программного кода не требуется.
* Изображения диаграмм (`.jpg`) открываются в браузере или встроенными средствами ОС.
* Техническая спецификация представлена в формате `.docx` и требует наличия Microsoft Word, Google Docs или LibreOffice Writer для просмотра.
* Исходники диаграмм в формате `.drawio` можно просматривать и редактировать через веб-сервис [diagrams.net](https://app.diagrams.net/) или в десктопном приложении Draw.io.

## 📈 Выводы по результатам проекта

В результате работы над проектом был подготовлен исчерпывающий пакет документации для интеграции новой функции фильтрации на платформе Otium. Построение UML Sequence Diagram позволило прозрачно отобразить синхронные и асинхронные вызовы между клиентом, сервером и базой данных, исключив архитектурные слепые зоны до старта разработки. Оформленный Dev Ticket перевел бизнес-требования в четкий технический язык, зафиксировав критерии приемки (Acceptance Criteria) и логику обработки ошибок, что минимизирует риски доработок на этапе тестирования.
