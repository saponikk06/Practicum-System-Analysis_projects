# 🚀 Портфолио: Системный анализ и проектирование (System Analysis Projects)

Добро пожаловать в мой репозиторий! 👋 Здесь собраны практические проекты и кейсы, демонстрирующие мои навыки в системном и бизнес-анализе, проектировании архитектуры программного обеспечения, моделировании баз данных и обеспечении качества (QA). 

🎯 Эти проекты покрывают полный жизненный цикл разработки: от сбора и документирования требований до построения ER-моделей, BPMN-схем и написания тестовой документации.

---

## 🗂 Структура репозитория

Репозиторий разбит на директории по конкретным проектам. Ниже приведено подробное описание каждой папки и её содержимого.

### 1. 🎓 `Academic-Works-SQL-DB`
**Описание:** Академические работы и отчеты, посвященные управлению данными и работе с реляционными базами данных (SQL).
* 📄 **`07_Academic-Report/`**: Содержит итоговую курсовую работу (`Course_Work_Data_Management_SQL.pdf`), описывающую подходы к проектированию БД, написанию SQL-запросов и управлению данными.

### 2. 🏨 `Capsule-Hotels-QA-Suite`
**Описание:** Комплексная документация по обеспечению качества (QA) и пользовательские инструкции для системы управления капсульными отелями.
* 🧪 **`05_QA-Documentation/`**: Тестовая документация. Включает детализированный тест-план (`Test_Plan_Capsule_v2.2.pdf`) и методологию программы тестирования (`Test_Program_Methodology_Capsule.pdf`).
* 📖 **`06_User-Docs/`**: Пользовательская документация (`User_Manual_Capsule.pdf`) с пошаговыми инструкциями по работе с системой.

### 3. 🔤 `Chatty-Vocabulary-App-Docs`
**Описание:** Спецификации требований (SRS) и UML-моделирование для мобильного приложения по изучению слов (флеш-карточки).
* 📋 **`01_Requirements-SRS/`**: Документы с требованиями к системе, включая общую спецификацию (`SRS_Chatty_v2.0.pdf`) и детализированное описание Use Case для функционала карточек (`Use_Case_Spec_Flashcards_Feature.pdf`).
* 📊 **`02_Modeling-UML-DFD/`**: UML-диаграммы (Use Case) и диаграммы словарей системы в форматах `.jpg` и исходниках `.drawio`.

### 4. 🛒 `Ecommerce-DB-Modeling-BPMN`
**Описание:** Моделирование бизнес-процессов (BPMN) и проектирование баз данных для сферы электронной коммерции и смежных систем.
* ⚙️ **`02_Modeling-UML-DFD-BPMN/`**: Схемы бизнес-процессов AS-IS/TO-BE в нотации BPMN. Включает процессы: оформление заказа, регистрация пользователя, запись пациента на прием, запрос на отпуск. 
* 🗄️ **`03_Data-Modeling-ERD/`**: Сущность-Связь (ER) диаграммы. Представлены в нотациях Чена (Chen) и "Вороньей лапки" (Crow's Foot). Включена схема нормализованной базы данных (до 2NF).

### 5. 🍿 `Otium-Streaming-Service-Analysis`
**Описание:** Анализ и техническое проектирование функционала для стриминговой платформы Otium.
* 🔄 **`02_Modeling-UML-DFD/`**: Sequence diagrams (диаграммы последовательности), описывающие взаимодействие компонентов системы при запросе списка контента.
* 💻 **`07_Technical-Specs/`**: Технические задания на разработку. Включает тикет для разработчиков на создание эксклюзивного фильтра (`Dev_Ticket_Exclusive_Filter_Spec.pdf`).

### 6. 🏠 `Stets-Home-Smart-System`
**Описание:** Глубокая аналитика и проектирование системы умного дома (Smart Home). Проект демонстрирует системный подход к интеграции железа и софта.
* 📚 **`01_Requirements-SRS/`**: Словарь данных системы (`Data_Dictionary_v2.pdf`).
* 🌊 **`02_Modeling-UML-DFD/`**: Диаграммы потоков данных (DFD) на контекстном и логическом уровнях.
* 🔗 **`03_Data-Modeling-ERD/`**: ER-диаграмма, построенная на основе DFD-контекста.
* 🎨 **`04_Design-Mockups/`**: Ссылки на макеты интерфейсов в Figma (`Figma_Mockups_Link.pdf`).
* 🐛 **`05_QA-Documentation/`**: Методология программы тестирования для системы умного дома.

## 🛠 Инструменты и технологии

В ходе работы над проектами я использовала следующий стек:
- 📝 **Анализ и требования:** SRS, Use Cases, User Stories, Data Dictionaries.
- 📐 **Моделирование процессов и архитектуры:** BPMN 2.0, UML (Sequence, Use Case), DFD (Контекстный и логический уровни).
- 🗃️ **Проектирование баз данных:** ER-диаграммы (Chen, Crow's Foot), Нормализация (1NF, 2NF, 3NF), SQL.
- 🔍 **QA и тестирование:** Разработка тест-планов, методологий тестирования.
- 🛠️ **Инструментарий:** Draw.io (diagrams.net), Figma, инструменты для работы с SQL.
