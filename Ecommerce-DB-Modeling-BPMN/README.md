# 🛒 Ecommerce DB & Process Modeling: Проектирование Базы Данных и Бизнес-Процессов

**Теги:** `BPMN 2.0` `ERD` `Data Modeling` `Business Process Modeling` `System Analysis` `Draw.io` `Database Normalization`

## 🚀 Обзор Проекта

Данный проект представляет собой коллекцию артефактов, демонстрирующих глубокую экспертизу в двух ключевых областях системного анализа: **Моделирование данных (Data Modeling)** и **Моделирование Бизнес-Процессов (BPMN)**.

Проект сосредоточен на разработке логических моделей для **Интернет-магазина**, включая структуру базы данных (ERD) и детализацию процессов взаимодействия с пользователем и операционной деятельности. Разнообразие представленных BPMN-диаграмм (от электронной коммерции до поликлиники) подчеркивает универсальность владения нотацией.

## 🎯 Ключевые Компетенции и Задачи

1.  **Моделирование Базы Данных (ERD):** Проектирование реляционной структуры, демонстрация нормализации и использование различных нотаций (Чена, "Воронья Лапка").
2.  **Моделирование Процессов (BPMN):** Детальное описание и визуализация бизнес-процессов (AS-IS и TO-BE), включая регистрацию пользователя и выполнение заказа.
3.  **Формализация Требований:** Создание точных, стандартизированных диаграмм, готовых к использованию разработчиками и бизнес-заказчиками.

## 🛠 Используемые Нотации и Инструменты

* **Моделирование Процессов:** BPMN 2.0 (Business Process Model and Notation)
* **Моделирование Данных:** ERD (Entity-Relationship Diagram), Нотация Чена, Нотация "Воронья Лапка" (Crow's Foot), Нормализация до 2NF
* **Инструментарий:** Draw.io (diagrams.net)

## ⚙️ Системные требования и инструкция по просмотру

Проект состоит из графических артефактов и их исходных файлов. Развертывание базы данных или программного кода не требуется.
* Изображения диаграмм (`.jpg`) можно просматривать стандартными средствами операционной системы или прямо в интерфейсе GitHub.
* Для просмотра, редактирования и экспорта исходных файлов диаграмм (`.drawio`) необходимо использовать веб-сервис [diagrams.net](https://app.diagrams.net/) или установить десктопное приложение Draw.io. Достаточно импортировать файл в среду разработки для продолжения работы.

## 📈 Выводы по результатам проекта

В результате работы над проектом были формализованы и визуализированы ключевые бизнес-процессы e-commerce площадки. Моделирование в нотации BPMN 2.0 позволило выявить участников процессов (покупатель, система, склад, курьер) и четко разграничить зоны их ответственности при оформлении заказов и регистрации. 

Проектирование базы данных сопровождалось приведением модели к 2-й нормальной форме (2NF), что обеспечило устранение частичных зависимостей и заложило фундамент для масштабируемой и непротиворечивой структуры данных интернет-магазина. Подготовленные артефакты полностью готовы для передачи технической команде для физической реализации БД и настройки backend-логики.
